I’ve talked about how the open source API search engine APIs.io can be deployed as an internal API search engine, as well as for public API discovery in the past.One aspect of this approach that I can’t emphasize enough, is that the search engine doesn’t just return APIs in the search results, it can provide access to supporting building blocks as well.The APIs.json format doesnt just provide a machine readable way to describe, and index your APIs, it also provides a way to describe the building blocks that support your APIs like code samples, SDKs, and other things API consumers will need when putting APIs to work for them.
